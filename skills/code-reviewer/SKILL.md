## $0 에 대한 코드 리뷰.

$0 가 파일이라면 파일에 대해 코드 리뷰, 코드블락이라면 주어진 코드 블락에 대해서 다음 지시사항을 따른다.

1. **라인별 코드 설명**: 각 줄마다 어떤 기능을 하는지, 기능상 무의미한 코드라면 삭제, 필요한 코드라면 설명 및 주석달기.
2. **코드 가독성 중시**: 코드 가독성을 최우선으로 할 것. 변수, 함수 이름 고려할 것. 불필요한 조건문, 반복문 지양.

## 코드 로직이 복잡한 경우에 대해서는 다음 항목에 대한 추가 설명 보충.

* **다이어그램 그려서 제공**: 각 로직의 기능을 다이어그램으로 제공하여 이해를 돕기.

### 코드 로직이 복잡한 기준

- 함수 호출 스택이 3개 이상 혹은 재귀함수인 경우.
- 함수 호출시 파라미터가 5개 이상인경우.
- if 문, 반복문이 3개이상인 경우.
